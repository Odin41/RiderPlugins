# Unused Methods Analyzer

Плагин для JetBrains Rider, который находит методы C# с нулевым количеством использований по всему решению — используя тот же движок поиска ссылок, что и **Find Usages** в Rider, расширенный текстовым поиском по разметке и коду для случаев, которые PSI пропускает.

---

## Возможности

- **Полное сканирование решения** или **только выбранной папки**
- **Трёхуровневое обнаружение ссылок** — без ложных срабатываний от вызовов через интерфейсы, обобщённых методов и привязок событий в Blazor/Razor
- **Сортируемая и фильтруемая таблица результатов** — одновременная фильтрация по имени метода, классу или файлу
- **Навигация одним кликом** — двойной клик или Enter для перехода к любому методу в редакторе
- **Контекстное меню по правой кнопке** — Go To, Mark \[Obsolete\], копирование имени / сигнатуры
- **Экспорт результатов** в CSV (совместим с Excel) или TXT (сгруппированный отчёт)
- **Сохранение результатов** — последний анализ автоматически восстанавливается при повторном открытии проекта
- **Маркеры в полях редактора** — файлы с неиспользуемыми методами отмечаются в gutter
- **Пометка как \[Obsolete\]** прямо из списка результатов с произвольным сообщением

---

## Как это работает

Анализ выполняется в три прохода:

**Проход 1 — Сбор кандидатов**
Обходит PSI-дерево всех файлов `.cs` и извлекает объявления методов. Имена извлекаются через трёхуровневый фоллбэк: интерфейс `PsiNamedElement` → reflection `getName()` → regex по тексту элемента. Конструкторы, деструкторы и сгенерированные файлы (`.Designer.cs`, `.g.cs`) исключаются автоматически.

**Проход 2 — Проверка каждого метода через три шага:**

| Шаг | Что проверяет | Что находит |
|-----|--------------|-------------|
| **A** PSI `ReferencesSearch` | Прямые семантические ссылки по проекту | Прямые вызовы, большинство использований |
| **B** Текстовый поиск по разметке | Файлы `.razor`, `.cshtml`, `.xaml` | `OnValidSubmit="Method"`, `@onclick="_ref.Method"`, `nameof(Controller.Method)` |
| **C** Текстовый поиск по C# | Все файлы `.cs` (regex по границам слова) | Вызов через интерфейс `_service.Method()`, обобщённые вызовы `GetError(arg)`, делегаты, `nameof`, строки рефлексии |

Метод попадает в отчёт как неиспользуемый только если все три шага не нашли ни одной ссылки.

**Обнаружение проектов**
Плагин парсит файл `.sln` и находит все каталоги `.csproj`, включая проекты вне `basePath` (общие библиотеки, вспомогательные проекты в соседних папках). Это гарантирует корректный анализ методов расширения и утилит из любого проекта в решении.

## Установка

1. Соберите плагин:
   ```bat
   docker-build.bat
   ```
   Файл `.zip` будет создан в `build/distributions/`.

2. В Rider: **Settings → Plugins → ⚙ → Install Plugin from Disk…** → выберите `.zip`.

3. Перезапустите Rider.

## Использование

### Анализ всего решения
- Нажмите **Analyze Project** в окне инструментов *Unused Methods*, или
- Используйте сочетание клавиш **Ctrl+Alt+Shift+U**, или
- Перейдите в **Tools → Analyze Unused Methods**

### Анализ отдельной папки
Правый клик на любой папке в дереве проекта → **Analyze Unused Methods Here**

### Навигация к методу
Двойной клик по строке, клавиша **Enter** или правый клик → **Go To**.

### Фильтрация результатов
Используйте три поля фильтрации в панели инструментов для отбора по имени метода, имени класса или имени файла. Все фильтры работают одновременно (логика AND). Кнопка **✕** сбрасывает все фильтры.

### Сортировка результатов
Кликните на заголовок колонки для сортировки. Повторный клик меняет направление. Третий клик снимает сортировку. Колонка **Line** сортируется численно.

### Пометка как \[Obsolete\]
Выберите метод → нажмите **Mark \[Obsolete\]** (или правый клик → **Mark \[Obsolete\]**) → введите сообщение. Атрибут вставляется над объявлением метода, метод удаляется из списка результатов.

### Экспорт
Нажмите **Export** в панели инструментов и выберите формат:
- **CSV** — разделённый запятыми, подходит для Excel и других табличных редакторов
- **TXT** — текстовый отчёт, сгруппированный по классам, подходит для код-ревью

## Настройки

Откройте **Settings** (иконка ⚙ в панели инструментов) для настройки:

| Параметр | Описание |
|----------|----------|
| \[Obsolete\] text | Текст сообщения по умолчанию при пометке через Mark \[Obsolete\] |
| Excluded names | Имена методов через запятую, которые всегда игнорируются |
| Exclude private methods | Пропускать методы с модификатором `private` |
| Exclude override methods | Пропускать методы с модификатором `override` |
| Exclude event handlers | Пропускать обработчики событий |
| Exclude test methods | Пропускать методы с атрибутами `[Test]`, `[Fact]`, `[Theory]` |

---

## Что исключается автоматически

Следующие методы никогда не попадают в отчёт вне зависимости от настроек:

- Конструкторы и деструкторы
- Методы с атрибутами: `[Obsolete]`, `[Test]`, `[Fact]`, `[Theory]`, `[UsedImplicitly]`, `[HttpGet/Post/Put/Delete/Patch]`, `[JSInvokable]`, `[Parameter]`, `[Inject]`
- Сгенерированные файлы: `*.Designer.cs`, `*.g.cs`, `*.g.i.cs`

---

## Известные ограничения

- **Область поиска PSI** ограничена тем, что `ReferencesSearch` IntelliJ может разрешить из Kotlin-фронтенда. Вызовы через строки рефлексии (например, `Type.GetMethod("MethodName")`) обнаруживаются только текстовым поиском — без семантической проверки.
- **Динамическая диспетчеризация** (например, вызовы через ключевое слово `dynamic`) не обнаруживается.
- Результаты **не обновляются автоматически** при редактировании кода — запустите анализ повторно после внесения изменений.

## Структура проекта

```
src/main/kotlin/com/github/unusedmethods/
├── CSharpAnalyzer.kt                 # Основной движок анализа (PSI + текстовый поиск)
├── UnusedMethodsToolWindowFactory.kt # UI окна инструментов, фильтры, экспорт
├── UnusedMethodsService.kt           # Сервис проекта, сохранение состояния
├── UnusedMethodsSettings.kt          # Настройки уровня приложения
├── UnusedMethodsGutterProvider.kt    # Маркеры в полях редактора
├── RunAnalysisAction.kt              # Действие для панели / сочетания клавиш
├── AnalyzeFolderAction.kt            # Действие контекстного меню дерева проекта
└── MethodInfo.kt                     # Data-класс для записи результата
```

## Требования к сборке

| Компонент | Версия |
|-----------|--------|
| JetBrains Rider | 2025.2+ |
| IntelliJ Platform Gradle Plugin | 2.11.0 |
