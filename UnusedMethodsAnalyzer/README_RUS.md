# Unused Methods Analyzer

Плагин для JetBrains Rider, который находит неиспользуемые символы C# по всему решению — методы, свойства, классы, интерфейсы и перечисления — используя тот же движок поиска ссылок, что и **Find Usages** в Rider, расширенный текстовым поиском по разметке и коду.

---

## Возможности

- **Полное сканирование решения** или **только выбранной папки**
- **Четыре типа символов** — методы, свойства, классы, интерфейсы и перечисления
- **Трёхуровневое обнаружение ссылок** — без ложных срабатываний от вызовов через интерфейсы, обобщённых методов и привязок событий в Blazor/Razor
- **Сортируемая и фильтруемая таблица результатов** — одновременная фильтрация по имени, классу, файлу и типу символа
- **Навигация одним кликом** — двойной клик или Enter для перехода к любому символу в редакторе
- **Контекстное меню по правой кнопке** — Go To, Mark \[Obsolete\], копирование имени / сигнатуры
- **Экспорт результатов** в CSV (совместим с Excel) или TXT (сгруппированный отчёт по типам символов)
- **Сохранение результатов** — последний анализ автоматически восстанавливается при повторном открытии проекта
- **Маркеры в полях редактора** — файлы с неиспользуемыми символами отмечаются в gutter
- **Пометка как \[Obsolete\]** прямо из списка результатов с произвольным сообщением (только для методов)

## Как это работает

Анализ выполняется в три прохода:

**Проход 1 — Сбор кандидатов**
Обходит PSI-дерево всех файлов `.cs` и извлекает объявления. Для каждого типа символа используется своя стратегия извлечения имени:

| Символ | PSI-тип | Извлечение имени |
|--------|---------|-----------------|
| Метод | `CSharpMethodDeclaration` | `PsiNamedElement` → рефлексия → regex метода |
| Свойство / поле | `CSharpPropertyDeclaration` | Regex свойства (`public T Name {`) |
| Класс / структура | `CSharpClassDeclaration` | Regex объявления типа (`class Foo`) |
| Интерфейс | `CSharpInterfaceDeclaration` | Regex объявления типа (`interface IFoo`) |
| Перечисление | `CSharpEnumDeclaration` | Regex объявления типа (`enum MyEnum`) |

Конструкторы, деструкторы и сгенерированные файлы (`.Designer.cs`, `.g.cs`) исключаются автоматически.

**Проход 2 — Проверка каждого символа через три шага:**

| Шаг | Что проверяет | Что находит |
|-----|--------------|-------------|
| **A** PSI `ReferencesSearch` | Прямые семантические ссылки по проекту | Прямые вызовы, большинство использований |
| **B** Текстовый поиск по разметке | Файлы `.razor`, `.cshtml`, `.xaml` | `OnValidSubmit="Method"`, `@onclick="_ref.Method"`, `nameof(...)` |
| **C** Текстовый поиск по C# | Все файлы `.cs` (regex по границам слова) | Вызов через интерфейс, обобщённые вызовы, делегаты, `nameof`, строки рефлексии |

Символ попадает в отчёт как неиспользуемый только если все три шага не нашли ни одной ссылки.

**Обнаружение проектов**
Плагин парсит файл `.sln` и находит все каталоги `.csproj`, включая проекты вне `basePath` (общие библиотеки, вспомогательные проекты в соседних папках).

## Установка

1. Соберите плагин:
   ```bat
   docker-build.bat
   ```
   Файл `.zip` будет создан в `build/distributions/`.

2. В Rider: **Settings → Plugins → ⚙ → Install Plugin from Disk…** → выберите `.zip`.

3. Перезапустите Rider.

## Использование

### Анализ всего решения
- Нажмите **Analyze Project** в окне инструментов *Unused Methods*, или
- Используйте сочетание клавиш **Ctrl+Alt+Shift+U**, или
- Перейдите в **Tools → Analyze Unused Methods**

### Анализ отдельной папки
Правый клик на любой папке в дереве проекта → **Analyze Unused Methods Here**

### Навигация к символу
Двойной клик по строке, клавиша **Enter** или правый клик → **Go To**.

### Фильтрация результатов
Используйте поля фильтрации в панели инструментов для отбора по имени, классу, файлу или типу символа. Все фильтры работают одновременно (логика AND). Выпадающий список **Type** позволяет показать только Methods, Properties, Classes, Interfaces или Enums. Кнопка **✕** сбрасывает все фильтры.

### Сортировка результатов
Кликните на заголовок колонки для сортировки. Повторный клик меняет направление. Третий клик снимает сортировку. Колонка **Line** сортируется численно.

### Пометка как \[Obsolete\]
Выберите метод → нажмите **Mark \[Obsolete\]** (или правый клик → **Mark \[Obsolete\]**) → введите сообщение. Атрибут вставляется над объявлением метода, метод удаляется из списка результатов. Доступно только для методов.

### Экспорт
Нажмите **Export** в панели инструментов и выберите формат:
- **CSV** — разделённый запятыми, содержит колонку `Kind`, подходит для Excel и других табличных редакторов
- **TXT** — текстовый отчёт, сгруппированный по типу символа, затем по классу, подходит для код-ревью

## Настройки

Откройте **Settings** (иконка ⚙ в панели инструментов) для настройки:

**Параметры методов**

| Параметр | Описание |
|----------|----------|
| \[Obsolete\] text | Текст сообщения по умолчанию при пометке через Mark \[Obsolete\] |
| Excluded names | Имена символов через запятую, которые всегда игнорируются |
| Exclude private methods | Пропускать методы с модификатором `private` |
| Exclude override methods | Пропускать методы с модификатором `override` |
| Exclude event handlers | Пропускать обработчики событий |
| Exclude test methods | Пропускать методы с атрибутами `[Test]`, `[Fact]`, `[Theory]` |

**Типы символов для анализа**

| Параметр | Описание |
|----------|----------|
| Analyze properties | Включить объявления свойств и полей |
| Analyze classes | Включить объявления классов и структур |
| Analyze interfaces | Включить объявления интерфейсов |
| Analyze enums | Включить объявления перечислений |

## Что исключается автоматически

Следующие символы никогда не попадают в отчёт вне зависимости от настроек:

- Конструкторы и деструкторы
- Методы с атрибутами: `[Obsolete]`, `[Test]`, `[Fact]`, `[Theory]`, `[UsedImplicitly]`, `[HttpGet/Post/Put/Delete/Patch]`, `[JSInvokable]`, `[Parameter]`, `[Inject]`
- Сгенерированные файлы: `*.Designer.cs`, `*.g.cs`, `*.g.i.cs`

---

## Известные ограничения

- **Область поиска PSI** ограничена тем, что `ReferencesSearch` IntelliJ может разрешить из Kotlin-фронтенда. Вызовы через строки рефлексии обнаруживаются только текстовым поиском — без семантической проверки.
- **Динамическая диспетчеризация** (вызовы через ключевое слово `dynamic`) не обнаруживается.
- **Извлечение имён классов/интерфейсов** основано на regex по тексту PSI-элемента, так как `CSharpClassDeclaration` не реализует `PsiNamedElement` в Rider. Если имя не найдено, в таблице будет `Unknown` — см. PSI dump в idea.log.
- Результаты **не обновляются автоматически** при редактировании кода — запустите анализ повторно после внесения изменений.

## Структура проекта

```
src/main/kotlin/com/github/unusedmethods/
├── CSharpAnalyzer.kt                 # Основной движок анализа (PSI + текстовый поиск)
├── UnusedMethodsToolWindowFactory.kt # UI окна инструментов, фильтры, экспорт
├── UnusedMethodsService.kt           # Сервис проекта, сохранение состояния
├── UnusedMethodsSettings.kt          # Настройки уровня приложения
├── UnusedMethodsGutterProvider.kt    # Маркеры в полях редактора
├── RunAnalysisAction.kt              # Действие для панели / сочетания клавиш
├── AnalyzeFolderAction.kt            # Действие контекстного меню дерева проекта
└── MethodInfo.kt                     # Data-класс для записи результата (с SymbolKind)
```
