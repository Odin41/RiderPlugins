# C# Namespace Converter – плагин для Rider

Преобразует **блочные пространства имён** в **файловые** (C# 10 / .NET 6+) одним кликом.

## Что делает

**До:**
```csharp
namespace MyApp.Services
{
    public class OrderService
    {
        public void Process() { }
    }
}
```

**После:**
```csharp
namespace MyApp.Services;

public class OrderService
{
    public void Process() { }
}
```

После преобразования плагин запускает встроенный форматтер Rider для исправления отступов.

## Установка (из исходников)

### Требования
- JDK 17+
- Gradle 8.5 (через wrapper — установка не нужна)
- JetBrains Rider 2023.3+ (или измените `version` в `build.gradle.kts`)

### Сборка

```bash
# Клонируйте / распакуйте проект, затем:
./gradlew buildPlugin
```

Готовый к установке `.zip` будет находиться по пути:
```
build/distributions/namespace-converter-1.0.0.zip
```

### Установка в Rider

1. Откройте Rider → **Settings** → **Plugins**
2. Нажмите иконку ⚙️ → **Install Plugin from Disk…**
3. Выберите собранный `.zip`
4. Перезапустите Rider

## Использование

Действие **«Namespace: Remove Curly Braces»** доступно в трёх местах для файлов `.cs`:

| Место | Как вызвать |
|-------|-------------|
| Дерево проекта | Правый клик на любом файле `.cs` |
| Вкладка редактора | Правый клик на вкладке файла вверху |
| Внутри редактора | Правый клик в любом месте кода |

Пункт меню **отображается только** если файл содержит блочное пространство имён — для файлов, уже использующих файловый синтаксис, он скрыт.

## Структура проекта

```
src/
  main/
    kotlin/com/github/nsconverter/
      ConvertNamespaceAction.kt   ← IntelliJ-действие (пункт меню, логика отображения)
      NamespaceConverter.kt       ← Логика преобразования (без зависимостей от IDE)
    resources/META-INF/
      plugin.xml                  ← Дескриптор плагина и регистрация действия
  test/
    kotlin/com/github/nsconverter/
      NamespaceConverterTest.kt   ← Юнит-тесты JUnit 5
build.gradle.kts
settings.gradle.kts
```

### Ключевые файлы

#### `NamespaceConverter.kt`
- **`hasBlockScopedNamespace(text)`** — проверка regex, используется для показа/скрытия пункта меню
- **`convert(text)`** — полное преобразование:
  1. Находит объявление `namespace Name` (без завершающей `;`)
  2. Находит соответствующую открывающую `{`
  3. Обходит файл, считая глубину вложенности скобок (пропуская строки, символы, комментарии)
  4. Находит закрывающую `}`
  5. Пересобирает файл: заменяет объявление на `namespace Name;`, убирает один уровень отступа у строк тела, удаляет внешние фигурные скобки

#### `ConvertNamespaceAction.kt`
- `update()` — выполняется в фоновом потоке; скрывает действие, если оно неприменимо
- `actionPerformed()` — запускает преобразование внутри `WriteCommandAction` (поддерживает отмену), фиксирует PSI, вызывает `CodeStyleManager.reformat()`, сохраняет документ

## Настройка версии Rider SDK

Отредактируйте `build.gradle.kts`:

```kotlin
intellij {
    type.set("RD")
    version.set("2023.3.2")   // ← укажите вашу версию Rider
}
```

Для использования **локальной установки Rider** вместо загрузки SDK:

```kotlin
intellij {
    type.set("RD")
    localPath.set("/path/to/JetBrains Rider 2023.3")
}
```

## Ограничения / известные проблемы

- Обрабатывает **одно пространство имён на файл** (стандартное соглашение C#).  
  Файлы с несколькими пространствами имён встречаются редко, но будет преобразовано только первое.
- Вызов `reformat()` использует активные настройки стиля кода C# в Rider, поэтому итоговые отступы автоматически соответствуют стилю вашего проекта.
